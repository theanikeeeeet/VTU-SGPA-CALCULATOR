# Contributing to VTU Calculator

Thanks for your interest in contributing! Small, simple steps:

1. Fork the repo and create a topic branch (git checkout -b feature/your-feature).
2. Make your changes and add tests if possible.
3. Keep changes focused and well-documented.
4. Run `npm test` and `npm run build` locally before opening a PR.
5. Open a PR targeting `main`. Describe the change and the motivation.

If you'd like help choosing an issue to work on, open an issue labeled `help wanted` or comment on an existing one.
